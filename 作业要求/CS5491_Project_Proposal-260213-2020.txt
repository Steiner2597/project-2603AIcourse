Published as a conference paper at ICLR 2024
A SAMPLE-EFFICIENT AND DIVERSITY-DRIVEN
LLM-GUIDED EVOLUTIONARY SEARCH FOR HEURIS TIC DESIGN
Wei LU Donghui YANG Wenzong DING
1 PROBLEM STATEMENT AND MOTIVATION
Problem. Heuristics are functions that guide search algorithms toward good solutions without guar anteeing optimality. Designing such heuristics by hand is time-consuming and requires substantial
domain expertise. We aim to automate heuristic design by using Large Language Models (LLMs)
inside an evolutionary search loop.
Application Domain. We study the Online Bin Packing Problem, where items arrive sequentially and must be packed into fixed-capacity bins to minimize the number of bins used. This NP-hard problem appears in logistics, cloud resource allocation, and memory management. Motivation. FunSearch showed that LLMs can discover useful programs by iteratively generating and evaluating code, but still suffers from redundant evaluations, high API cost, and premature convergence. Our project, FunSearch-Lite, targets these issues in a course-scale system.
2 PROPOSED METHOD
We propose FunSearch-Lite, an enhanced FunSearch-style system with three main ideas:
Innovation 1: Functional Deduplication (Sample-Efficient). We introduce a new deduplication
mechanism that combines normalized code hashing and behavior-based signatures, so that function ally equivalent candidates are evaluated only once.
Innovation 2: Multi-Model Collaboration (Cost-Aware). We separate exploration and refinement: a cheaper model generates many candidates, while a stronger model is reserved for refining only the most promising ones, aiming to improve search quality under a fixed API budget.
Innovation 3: Diversity-Driven Search (Novelty-Oriented). We maintain multiple evolving “is lands” and use behavior-based diversity signals to keep candidates with different behaviors, reducing the risk of getting trapped in local optima.
3 EVALUATION PLAN
Datasets. We plan to evaluate on two types of benchmark instances:
• OR-Library Instances: Standard bin packing test sets (Falkenauer’s uniform and triplet instances) with known optimal solutions, enabling comparison with published results.
• Synthetic Instances: Randomly generated instances for rapid iteration and ablation stud ies.
Baselines. We will compare against: (1) First-Fit (FF), a simple greedy heuristic; (2) Best-Fit(BF), which minimizes wasted space; (3) Ablated variants of our system to isolate each innovation's contribution.
Metrics. Primary: total bins used, gap to best-known solutions. Secondary: deduplication rate, APIcost savings, population diversity index, and convergence speed.
Ablation Studies. We plan to isolate the contribution of each innovation by running variants: (A)Full system; (B) Without functional deduplication; (C) Without multi-model collaboration; (D)Without diversity mechanisms.
